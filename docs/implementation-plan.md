## 運用ルール

- 機能を実装したら、該当項目を `- [x]` に変更する。
- 機能追加コミットでは、必要に応じて `docs/requirements.md`、`docs/manual.md`、このファイルを同時に更新する。`docs/PDF/manual.pdf` は最終版でまとめて生成する。
- `scan-001.pdf` と `scan-001 (1).pdf` は絶対にコミットしない。
- 参照PDFを忠実に再作図できることを最優先の受入条件とする。
- `C:\Users\oltot\Documents\git-projects\Autocad\dist\index.html` の端子座標モデルと配線追従ロジックを参考にし、現在の単一HTMLへ必要な考え方を取り込む。
- トヨタ標準対応は仕様確定後に差し替え/追加できるよう、標準パック、図枠、記号、線番、帳票、監査を分離して拡張する。
