## 1. 実装内容サマリ

### 1.1 ジオメトリ単一定義方式(第1部2章)
- `SYMBOL_GEO` レジストリ+ `drawGeoSymbol`(SVG)/ `dxfGeoSymbol`(DXF)/ `geoAnchors`(接続点)を追加。
  1つの定義から3系統を導出するため、SVGとDXFの乖離は構造的に発生しない。
- `drawElement` / `dxfElement` / `elementConnectionAnchors` の先頭でジオメトリ定義を優先ディスパッチ。
- 旧 `drawXxx` / `dxfXxxLocal` は残置(未到達。削除していない)。
- DXF に ARC エンティティ出力(`dxfArcLocal`)、回転文字(`dxfTextRotated`)、
  2次ベジェ近似(`dxfQuadApproxLocal`)を追加。

### 1.2 スキャン実測形状に置換したシンボル(第1部4章)
contactNO/contactNC(縦バー2本+斜線)、thermalRelay(○✕○)、breaker(円2+上アーク+段差トリップ、
`refNo` で定格表示)、coil(無印円・下に品番=catalog表示)、lamp(円+45°ヒゲ4本+`lampColor`色文字)、
pushButton(`pbStyle`: 標準=上浮きブレード+突起 / 非常停止=キノコ+戻し弧+R)、selectorSwitch(フック持ち手)、
limitSwitch(円2+上の横長矩形)、motor(円内タグ+左小円端子3+右下接地スタブ)、ground(縦線+斜めハッチ3本)、
terminal(⌀1.8小円のみ)、resistor(`resistorStyle`: 枠/バリスタ斜線)、connector(`connStyle:"pair"` でプラグ+ソケット弧)、
contactBlock(実線枠+内部バー接点、`blockFrame:"dashed"` で点線枠、NC斜線対応)。

新タイプ追加: `thermalElement`(⊓形)/ `pressureSwitch`(円2+斜めブレード+ドーム)/
`positionMark`(入/切ポジション、`markStyle`)/ `polarityMark`(⊕⊖、`sign`)。
PALETTE・タグ接頭辞(OL/PS)・中心配置・配線自動分割対象へ登録済み。
**全シンボルは2段ラベル対応**: description の1行目が機能説明としてタグの上に小さく出る(`descLine`)。

### 1.3 図枠「設備標準」(`frameVariant:"ladder"`、第1部5章)
- 行番号00〜20(9.5mmピッチ、行00=Y27)、外枠1本線、ページ名ヘッダ中央。
- 左下: クロスリファレンス表(13列)/右下: 機器符号表(2グループ×3列×10行)+表題欄
  (客先機番・装置区分・内容・頁・図番)+下部余白行(汎用図番/図番/出力日/更新日)。
- 左余白の縦書き側注記 `page.title.sideNote`(右パネル「側注記」で入力)。
- **DXFにも同じ図枠を出力**(`dxfLadderFrame`)。
- 右下組織名はハードコードせず `state.project.designer` があれば表示(スキャン文言排除ルール)。

### 1.4 クロスリファレンス自動化(第2部2章・実機検証済み)
- アドレス体系 = **頁番号+行番号2桁**(例: 頁2 行05 → `205`)。`LADDER_GRID`(top27/pitch9.5)基準。
- コイル(親)と接点(子)を全ページからタグで対応付け(`buildCrossRefCache`、renderごとに再計算)。
- 接点の下の参照番号は**親コイル位置を自動表示**(手入力 `refNo` が優先)。
- 図枠左下のクロス参照表に、そのページのコイルごとの**子接点アドレス一覧を自動記入**
  (b接点使用は**数字に下線**)。機器符号表にはページ上の機器タグ+説明+親コイル頁を自動記入。
- クロス参照ダイアログに「コイル→接点」ビューを追加(アドレスボタンで該当要素へジャンプ)。

### 1.5 端子台・端子配線セットの見本準拠化(作業中のユーザー指示対応)
- 端子台 `terminalStrip` に **`stripStyle:"unit"`(既定)** を追加:
  幅10.5mm縦カラム+上下波形破断線+⌀7端子円+円内番号。接続アンカーは**左縁**。
  旧ピル形は `stripStyle` 空で維持(旧データはそのまま旧形状で表示)。
- 端子配線セット(パレット/端子図テンプレート)を全面書き換え:
  横配線(線番は**終点寄せ**)→ 端子小円 → ユニット列端子台、右に信号名。
- 端子図テンプレートは ladder 図枠+P1/N1レール+接続点つきで生成。

### 1.6 テンプレート/ビルダー(第1部6章+第2部3章)
- **設備ラダー**: レール名CSV(複数レール対応、右から4mmピッチ・ラベル階段配置)、
  ⊕⊖極性マーク、フォーク矢印(上下)、(DC24V)注記。
- **PLC入力回路**: ユニット名/開始アドレス/点数 → **16進連番**(X000→…→X00F)で
  レール+接点+配線+端子小円+ユニット列+行アドレス整列を一括生成。
- **PLCユニット**単体生成 / **モータ分岐回路**(CB3極+連動破線+MC接点+サーマル素子+モータ+接地、
  配線は装置端子で分割済み)/ **ツイストペア**(楕円束マーク+X交差+破線シールド+シェブロン)。

### 1.7 ACE機能(第2部3〜4章)
- **部品カタログDB**(`state.catalog`): 管理ダイアログ+「カタログから選択」(電気属性欄)。
  a/b接点上限(maxNO/maxNC)を保持。
- **BOM**: メーカー×品番×種別で数量集計。CSV出力+表として挿入。
- **監査強化**: 接点なしコイル(情報)/コイルなし接点(注意、OL単独は除外)/
  接点数上限超過(警告、カタログ値または既定4)/ジャンパ番号範囲外(警告)/
  盤レイアウト未配置(フットプリント使用時のみ・情報)。
- **タグ重複のリアルタイム警告**(タグ入力確定時にステータス表示)。
- **端子台ジャンパ** `jumpers:"1-2;3-5"`: 描画(弧)+DXF+右パネル+端子台エディタ列。
- **盤レイアウト連携**: タグ付き部品をチェック選択→フットプリント矩形(`panelRef`)を格子挿入。
- **標準パック**: JSON出力/取込(`packVersion:1`)。取込パックは標準セレクトに追加され即適用。
- **DXF取込MVP**: LINE/CIRCLE/TEXT/LWPOLYLINE → line/circle/text 要素(Y反転済み)。
- **線番書式 `%R`**(ラダー行)を線番・部品タグ書式に追加(`%S%R` で頁+行形式が可能)。
- **編集操作**: グループ化/解除(クリックでグループ全選択)、左右ミラー、等間隔複製(既定Y=9.5)、
  配線の結合(2本・端点一致)/分割(中点)。
- **配線ラベル配置**: `wireNoAlign`(中央/始点/終点)+`wireNoDx/Dy` 微調整(SVG/DXF両対応)。
- **信号矢印**: `arrowStyle`(open/solid/fork/chevron)+`arrowAtStart`。
  source/destination ペアが揃うと**相手の頁番号を自動注記**。
- **線種プリセット**(直線): 実線/破線/一点鎖線/二点鎖線/細点線。文字の**下線**(`underline`)。
- **表の斜線セル** `cellDiagonals:"行,列;行,列"`(SVG/DXF)。
- **plcBlock `plcStyle:"unit"`**: 縦ユニット列(波形破断線+⌀7円)。格子形は既定のまま維持。

## 2. 仕様書からの逸脱・判断(重要)

1. **terminal のアンカー順**を [左,中央,右] に変更し(仕様書は[中央,左,右])、`terminal` を
   配線自動分割対象に追加。分割ロジックが先頭/末尾アンカーを使うため。
2. **設備ラダーの既定レール**は `P1` 1本(第2部2.9の既定4本はプレースホルダに変更。
   固有名 P3M はスキャン由来のため使用しない)。
3. レールのフォーク矢印は短い arrow 要素(`ladderRail` フラグ付き)で実装し、
   **監査の「矢印ラベル未入力」警告から除外**(レール矢印で監査が埋まるのを防止)。
4. arrow の signalRole 文字は従来どおり**図面上に表示しない**(第1部4.18のサンプルコードには
   あったが、ペア注記(頁N)で代替。必要なら drawArrow 末尾に1行追加で出せる)。
5. PLC入力テンプレートは**説明テキスト列を生成しない**(空文字要素が「文字」と表示される既存仕様を
   回避)。接点ラベルも空で生成 → 監査で「ラベル未入力(注意)」が出るのは正常動作。
6. geo シンボルは**角ハンドルでのリサイズ不可**(固定寸法のため。`canResizeElement` で除外)。
   w/h 数値入力は可能で、その場合ジオメトリは枠内中央に配置される(旧データ互換のための中央合わせ)。
7. `junction` の描画フォールバック半径を 1.45→0.75 に変更(旧データで r 未保存のものも新径になる)。
8. contactBlock の点線枠は `blockFrame:"dashed"` を geoPrims で外枠rectに注入する方式
   (定義4分割ではなく2定義+動的dash)。
9. 盤レイアウト未配置の監査は「フットプリントが1つ以上あるとき」だけ動く(第2部の「任意」をゲート実装)。
10. supportsElectricalAttributes の除外に positionMark / polarityMark / mechanicalLink を追加
    (補助記号が帳票・BOM・属性UIに混ざらないように)。

## 5. 既知の制限・残課題

1. ladder図枠のレイアウトは**A4縦前提**(横/A3では下部表が行番号帯と重なる)。
2. DXFの点線(接点ブロック点線枠・一点鎖線)は実線で出力(LTYPE未定義)。
3. ミラーはシンボル内部形状を反転しない(境界基準の位置反転。見本記号はほぼ対称のため実用上可)。
4. 配線分割は2点配線=中点/多点=中央頂点のみ(任意点クリック分割は未対応)。
5. DXF取込は LINE/CIRCLE/TEXT/LWPOLYLINE のみ(ARC/INSERT/寸法は対象外)。
6. クロス参照表は1ページ13コイル・1コイル12接点まで表示(見本と同じ枠数)。
7. `docs/manual.md` `docs/implementation-plan.md` `docs/scan-reproduction-checklist.md` は
   **未更新**(本タスクは index.html のみの指示のため)。次回、今回分の機能追記が必要。
8. 検証用に一時起動したローカルサーバー(127.0.0.1:8765)は停止済み。通常どおり
   Live Server(5502)または直接ファイルで開いてよい。
